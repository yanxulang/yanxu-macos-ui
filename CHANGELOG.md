# 更新记录

## 0.8.5 - 2026-07-18

- 滚动容器支持 `显示滚动指示器（值）`，并透传到 SwiftUI 的 `showsIndicators`；可在弹出层中保持滚动能力而不显示闪现的滚动条。
- 保持菜单栏弹出层顶部对齐、API schema 和原生 ABI 不变；不包含任何应用专属业务逻辑。

## 0.8.4 - 2026-07-18

- 修复菜单栏弹出层宿主的根视图对齐：内容现在填满弹出层并固定在顶部，避免固定尺寸较大时内容被压到底部留下空白。
- 保持菜单栏 API、schema 和原生 ABI 不变；不包含任何应用专属业务逻辑。

## 0.8.3 - 2026-07-18

- 新增通用 `application.reopened` 生命周期事件，携带 `hasVisibleWindows`，让纯言序应用区分登录启动后的用户再次打开行为。
- 保持窗口、菜单栏、设置和请求 API 不变；未加入任何应用专属业务逻辑。

## 0.8.2 - 2026-07-17

- 迁移到言序 1.1.14 的模块限定类型规则，所有跨模块公开类型均使用稳定的限定身份。
- 公共门面公开重导出视图、状态、动作、应用、运行时、请求与修饰器模块，避免公共签名泄露私有导入类型。
- 更新公开版本常量与测试断言；Swift/AppKit 宿主、原生 ABI v2 和动态库制品保持不变。

## 0.8.1 - 2026-07-17

- Keychain 与定时器请求的完成事件改为下一轮主循环投递，避免在言序事件处理器内同步重入 callback pump。

## 0.8.0 - 2026-07-17

- 移除通用 HTTP 与系统时钟请求；网络和日期时间能力改由纯言序包提供。
- 为文本与安全文本输入补齐标准编辑菜单，恢复剪切、复制、粘贴、全选及其键盘快捷键。
- 保留菜单栏宿主边界内的系统钥匙串和非阻塞重复定时器能力。

## 0.7.4 - 2026-07-17

- 嵌套事件回调已成功入队时，不再把所有者线程正在泵送误判为投递失败并终止应用。
- 启动完成前收到停止请求时不再进入 AppKit 事件循环。

## 0.7.3 - 2026-07-17

- `退出` 现在请求 AppKit 真正终止应用进程，不再只停止当前事件循环，修复纯菜单栏应用退出后仍驻留的问题。

## 0.7.2 - 2026-07-17

- 表单改用适合菜单栏弹出层的单列全宽布局，避免原生双栏 `Form` 的固有尺寸将输入框和 Stepper 推出可视区域。

## 0.7.1 - 2026-07-17

- 时钟请求支持秒级偏移，纯言序应用可构造精确的短周期 API 时间范围。

## 0.7.0 - 2026-07-17

- 新增通用异步 HTTP 请求、系统钥匙串安全存储和重复定时器请求。
- Stepper 标签现在显示绑定的当前数值，设置项可直接确认与调整。
- 补充通用请求的类型检查覆盖和 API 文档。

## 0.6.0 - 2026-07-17

- 新增 `麦金塔菜单栏项目`：通过 `NSStatusItem` 在 macOS 菜单栏显示 SF Symbol，点击后打开承载任意已支持 SwiftUI 视图的 `NSPopover`。
- 支持没有主窗口的纯菜单栏应用；宿主使用 accessory activation policy，不在 Dock 显示应用图标。
- 菜单栏弹出层复用 schema v2 状态与 Binding，可包含文本、按钮、输入、选择器、列表及其他组合视图。
- 新增 `退出（）` 原生入口，纯菜单栏应用可从言序事件处理器正常结束应用会话。
- 完整快照更新支持动态增加、更新和移除菜单栏项目，并保持弹出层内容响应状态 patch。
- 新增 `麦金塔场景` 继承树与 `麦金塔运行会话`：窗口、设置、菜单栏和文档成为具体场景对象，会话负责挂载、revision、请求关联与退出。
- 窗口场景支持稳定 ID、初始显隐、动态打开/关闭、frame 恢复以及应用和窗口生命周期事件；设置场景接入标准 Settings 菜单和原生窗口。
- 命令和工具栏新增 placement、角色、SF Symbol、默认命令及布尔 Binding 校验，可随焦点和业务状态动态启用。
- 新增带独立状态 Store 的多文档窗口模板和标准 New/Open/Save 菜单，以及打开、保存、导入、导出和文档请求；异步结果按请求 ID 回传文件内容、编码和 security-scoped bookmark。
- 新增 NavigationStack 与绑定路径、Table 行列对象/选择/字段排序、Outline 节点对象、Disclosure、ControlGroup、焦点 Binding 和 inspector；macOS 13 的 inspector 使用分栏降级。
- 新增完整桌面应用示例，并将新增协议、场景、请求和视图能力纳入 Swift 与言序回归检查。

## 0.5.0 - 2026-07-17

- 新增文字、数值、布尔和选择状态对象，以及对应的类型化 Binding；输入事件会在用户处理器执行前写回言序状态。
- 应用协议升级到 `dev.yanxu.mac-ui.v2`，加入稳定状态表和 revision；原生 `patch` 入口支持高频状态增量更新，同时保留 schema v1 解码兼容。
- 新增绑定文本、绑定进度、Picker、Slider、Stepper、DatePicker、ColorPicker、搜索、可选择列表、Label、Link 和 Menu。
- 新增状态驱动的 Sheet、Popover 和 Alert，并把应用强调色应用到 SwiftUI 视图树。
- v2 挂载校验重复状态/视图 ID、未知 Binding 和类型不匹配；状态 patch 不允许增删或改变状态类型，过期 revision 不会覆盖新值。
- 将 Swift renderer 拆为 Store、控件、布局、集合、呈现和样式模块；计数器示例改为无需解析输入载荷或提交完整快照的状态驱动实现。

## 0.4.0 - 2026-07-17

- 将公开视图模型重构为具体类层次，文本、按钮、输入控件、容器和基础视图不再全部返回万能 `麦金塔视图`。
- 新增 `麦金塔动作`，按钮、菜单、工具栏和输入控件可以共享同一个类型化用户意图对象；字符串事件名继续兼容。
- 新增 `麦金塔视图修饰器` 及内边距、字体、框架修饰器对象，保留 `设样式` 作为低层扩展入口。
- 新增控件专用方法，包括 `设内容`、`设值`、`当点击`、`当文本变化`、`添视图` 和 `设间距`。
- 将任意 `kind` 构造隔离到 `麦金塔自定义视图`，传输层 JSON schema 保持 `dev.yanxu.mac-ui.v1` 兼容。
- 将单体源文件拆为基础、动作、修饰器、视图、应用模型、运行时和稳定公共门面，保持现有工厂 API 与静态返回类型。

## 0.3.0 - 2026-07-17

- 新增应用级 `当事件` 注册，ABI v2 宿主按 retain/post/pump/release 生命周期把按钮、菜单、工具栏和输入事件送回言序 VM。
- 新增 `更新（应用）` 快照协议，运行中的窗口、SwiftUI 内容、菜单和工具栏可由言序业务处理器原位更新。
- 文本框、安全文本框、多行文本和开关改为真实状态绑定，`设事件` 载荷包含 `source` 与类型化 `value`。
- 视图模型改用通用 property bag，扩展属性不再要求修改固定解码 schema。
- 计数器改为完整交互示例，并新增 ABI 回调与状态更新 Swift 回归测试。

## 0.2.0 - 2026-07-15

- 将 `运行（应用）` 切换为言序 1.1.7 `标准:原生` + ABI v2 调用，不再启动外部进程。
- Swift 宿主新增 `yanxu_native_module_v2`，保留 ABI v1 入口以兼容旧验证工具。
- 清单升级到 `ABI = 2`，记录麦金塔 arm64 制品大小与 SHA-256。
- 文档更新为顶层应用授权 `原生扩展 = true` 的正式运行方式。

## 0.1.5 - 2026-07-15

- 随包发布麦金塔 arm64 原生动态库制品，并在清单中声明 SHA-256 校验和。
- 将文档更新为“应用作者只写言序代码，普通安装不需要手动配置 runner PATH”。
- 保留 runner 作为宿主开发和 1.1.x 过渡预览工具。

## 0.1.4 - 2026-07-15

- 新增 `设运行器（路径）` 和 `YANXU_MACOS_UI_RUNNER` 支持。
- runner 采用脱离启动模式，言序程序无需等待窗口关闭。
- 示例改为直接调用 `界面.运行（应用）`。

## 0.1.3 - 2026-07-15

- 新增 Swift Package 原生宿主骨架，使用 SwiftUI 渲染视图并由 AppKit 管理窗口。
- 加入示例、测试、API 文档和安全说明。
