# 入门

`yanxu-macos-ui` 的核心思路很简单：用言序写出应用的骨架，用事件名连接业务逻辑，用原生宿主把它呈现为麦金塔应用。

应用作者只写言序。SwiftUI 和 AppKit 位于包内原生宿主层，负责呈现窗口、控件和系统交互，不要求业务项目包含 Swift 源码。

## 安装

```sh
yanbao add macos-ui
```

`macos-ui` 会解析到 `yanxulang/yanxu-macos-ui`。需要固定版本时可追加版本约束，例如 `--version '^0.5'`。

或者在本仓库中直接引用源码：

```yanxu
引「../src/言序麦金塔界面.yx」为 界面；
```

## 一个窗口

```yanxu
定 下一步 为 界面.动作（「next」）；
定 根 为 界面.垂直（【
    界面.标题（「第一扇窗」）,
    界面.文本（「言序描述应用，宿主负责原生呈现。」）,
    界面.按钮（「继续」，下一步）
】）；
根.内边距（20）；

定 窗口 为 界面.窗口（「示例」，根）.尺寸（860，560）；
```

## 一个应用

```yanxu
定 应用 为 界面.应用（「示例」）
    .设强调色（「teal」）
    .添窗口（窗口）；

法 处理事件（所事件）则
    若 所事件.名称 等于「next」则 言「继续」；终
终

应用.当事件（处理事件）；
界面.运行（应用）；
```

应用必须至少有一个窗口。事件名必须是稳定 ASCII 标识，例如 `draft.save`、`build.start`，这样原生层、日志和测试都能可靠识别。

## 加菜单和快捷键

```yanxu
定 文件菜单 为 界面.菜单（「文件」）
    .添命令（界面.命令（「保存」，「draft.save」）.设快捷键（「s」，【「command」】））；

应用.添菜单（文件菜单）；
```

菜单会挂入系统菜单栏，并与按钮、工具栏一样进入应用事件处理器。

## 创建菜单栏图标

0.6 支持纯菜单栏应用。`菜单栏项目` 使用 SF Symbol 作为图标，点击后显示可包含任意已支持控件的 SwiftUI 弹出层：

```yanxu
定 内容 为 界面.垂直（【
    界面.文本（「后台任务正在运行」），
    界面.按钮（「退出」，「app.quit」）
】）；

定 应用 为 界面.应用（「后台工具」）
    .添菜单栏项目（界面.菜单栏项目（「main-status」，「bolt.fill」，「后台工具」，内容））；

法 处理事件（所事件）：空 则
    若 所事件.名称 等于 「app.quit」 则 界面.退出（）；终
    归 空；
终

应用.当事件（处理事件）；
界面.运行（应用）；
```

纯菜单栏应用不会显示 Dock 图标。若应用还注册了设置或辅助窗口，可在 `应用（名称）` 后调用 `.设程序坞图标（假）`，在保留窗口能力的同时继续作为无 Dock 图标的菜单栏应用运行。完整示例见 `examples/菜单栏工具.yx`。

## 更新界面

0.5 起，业务值优先使用状态和 Binding。输入变化会先写回言序对象，处理器返回后自动提交 revision patch：

```yanxu
定 标题状态 为 界面.文字状态（「document.title」，「未命名」）；
定 标题输入 为 界面.绑定文本框（「title」，「标题」，标题状态.绑定（））；

定 应用 为 界面.应用（「编辑器」）
    .添状态（标题状态）
    .添窗口（界面.窗口（「编辑器」，标题输入））；
```

处理器可以直接调用 `标题状态.取值（）`。只修改状态时不需要 `界面.更新`；在事件处理器之外主动修改状态后，可调用 `界面.提交状态（应用）`。

### 结构更新

处理器收到 `麦金塔事件`，其中 `名称` 是稳定动作名，`载荷` 至少包含 `source`。文本框可以用 `.当文本变化（动作）` 注册变更；其他输入控件继承 `.当变化（动作）`，载荷还包含当前 `value`。

修改原有视图或窗口对象后提交新快照：

```yanxu
法 处理事件（所事件）则
    若 所事件.名称 等于「title.changed」则
        标题视图.设内容（所事件.载荷【「value」】）；
        界面.更新（应用）；
    终
终
```

`更新` 不重启应用；宿主保留窗口并让 SwiftUI 从新快照重渲染。它用于窗口、菜单和视图树结构变化，不应代替高频状态 patch。

## 打开窗口

示例末尾调用 `界面.运行（应用）`。包随附麦金塔原生宿主制品，清单记录目标平台、文件路径、大小和 SHA-256；应用作者不需要构建 Swift，也不需要配置外部运行器。

顶层应用需要允许原生扩展：

```toml
[权限]
原生扩展 = true
```

宿主开发者调试源码时，可以本地构建 Swift Package 并重新生成制品校验和：

```sh
swift build -c release --package-path native
shasum -a 256 native/.build/release/libYanxuMacUIHost.dylib
```

## 原生宿主源码

```sh
swift build -c release --package-path native
```

当前包声明麦金塔 arm64 ABI v2 原生制品，安装时不会要求用户构建 Swift。仓库保留宿主源码和 v1 runner，用于维护与旧调试路径。

本地构建产物通常位于：

```text
native/.build/release/libYanxuMacUIHost.dylib
```
