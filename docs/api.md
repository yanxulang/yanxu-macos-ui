# 公开 API

本文记录言序侧稳定入口。内部 JSON 字段保持英文，是为了与 Swift 原生宿主和跨语言工具可靠通信；公开给言序作者的 API 使用中文。

## 常量

- `版本`：当前库版本。
- `名称`：`言序麦金塔界面`。
- `原生宿主名`：`yanxu-macos-ui`。

## 应用

`应用（名称）` 创建 `麦金塔应用`。

常用方法：

- `设强调色（颜色）`
- `添窗口（窗口）`
- `添场景（场景）`
- `添状态（状态）`
- `添菜单（菜单）`
- `添菜单栏项目（项目）`
- `添菜单项（标题，事件名）`
- `设设置（视图）`
- `设设置场景（场景）`
- `添文档场景（场景）`
- `设文档型（值）`
- `当事件（处理器）`
- `描述（）`
- `JSON（）`

`JSON（）` 要求至少包含一个窗口、菜单栏项目或文档场景，因此纯菜单栏和纯文档应用都是有效应用。

`运行（应用）` 通过 `标准:原生` 装载已校验的 ABI v2 宿主，并启动窗口或菜单栏项目。调用方顶层项目需要允许 `原生扩展 = true`。`更新（应用）` 提交结构快照；`提交状态（应用）` 只提交状态表。事件处理器返回后会自动提交状态，通常不需要手工调用后者。

推荐由 `会话（应用）` 创建 `麦金塔运行会话`，再调用 `会话.运行（）`。会话对象提供 `提交状态（）`、`更新（）`、`执行请求（请求）` 和 `退出（）`，并按请求 ID 将 `request.completed` / `request.failed` 路由给请求对象。全局函数继续作为兼容门面。

## 状态与 Binding

`文字状态`、`数值状态`、`布尔状态`、`选择状态` 分别创建四种具体状态。每个状态提供类型匹配的 `取值（）`、`设值（值）` 和 `绑定（）`。状态必须通过 `应用.添状态` 注册，状态 ID 在应用内唯一；Binding 只保存状态 ID 和类型，不复制值。

原生输入先写回状态，再调用事件处理器，因此业务代码可以直接读取状态对象：

```yanxu
定 标题状态 为 界面.文字状态（「document.title」，「未命名」）；
定 标题输入 为 界面.绑定文本框（「title-field」，「标题」，标题状态.绑定（））；
```

## 窗口

`窗口（标题，根视图）` 创建兼容窗口；`窗口场景（标识，标题，根视图）` 创建具有稳定场景 ID 的 `麦金塔窗口`。

常用方法：

- `尺寸（宽，高）`
- `最小尺寸（宽，高）`
- `设可调整大小（值）`
- `设默认命令（事件名）`
- `添工具项（标识，标题，事件名）`
- `添工具栏项（工具栏项）`
- `设场景标识（标识）`
- `设恢复标识（标识）`
- `设初始显示（值）`
- `描述（）`

尺寸必须在 `0..16384` 范围内。

`工具栏项（标识，标题，动作）` 创建 `麦金塔工具栏项`，支持 `设系统图标`、`设位置` 和 `绑定启用`。位置包括 `automatic`、`navigation` 和 `primary`。

`设置场景（内容）` 创建原生设置窗口场景，可用 `尺寸` 配置窗口。宿主自动安装标准 Settings 菜单项。

`文档场景（标识，标题，内容类型，根视图）` 创建多文档窗口模板，内容类型使用 UTType 标识。宿主安装可直接工作的 New、Open 和 Save 菜单；`设默认文件名` 和 `尺寸` 配置新文档，`绑定内容（文字绑定）`、`绑定路径（文字绑定）` 指定打开文件时初始化的状态。每个文档窗口持有独立 Store，编辑不会直接覆盖其他文档实例。

## 菜单与命令

`菜单（标题）` 创建 `麦金塔菜单`。

`命令（标题，动作）` 创建 `麦金塔命令`。`动作` 可以是 `麦金塔动作`；0.3 的字符串事件名继续兼容。

命令方法：

- `设快捷键（键，修饰键）`
- `设角色（角色）`
- `设标识（标识）`
- `绑定启用（布尔绑定）`
- `绑定状态（布尔绑定）`

`菜单.设位置（位置）` 表达命令组 placement。工具栏和菜单会在显示前从 Store 校验启用 Binding；默认命令在窗口收到 Return 时回传并带窗口上下文。

事件名必须是稳定标识，推荐使用 `领域.动作`，例如 `project.open`、`draft.save`。

## 菜单栏项目

`菜单栏项目（标识，系统图标，提示文字，内容）` 创建 `麦金塔菜单栏项目`。系统图标使用 SF Symbols 名称；点击图标后显示 SwiftUI 弹出层，`内容` 可以是任意当前 renderer 支持的视图树。

常用方法：

- `尺寸（宽，高）`
- `设系统图标（系统图标）`
- `设提示文字（提示文字）`
- `描述（）`

弹出层最小尺寸为 `160 x 120`。菜单栏项目与窗口共享应用状态、Binding、事件处理器和 revision patch。

## 对象化视图

视图工厂返回具体类，而不是统一返回属性容器。主要继承关系：

- `麦金塔视图`：共同树结构、修饰和序列化行为；
- `麦金塔文本视图`：提供 `设内容`；
- `麦金塔交互视图`：提供 `当触发`；
- `麦金塔按钮视图`：提供 `设标题` 和 `当点击`；
- `麦金塔输入视图`：提供 `当变化`；
- `麦金塔文本框视图`、`麦金塔多行文本视图`、`麦金塔开关视图`：提供类型匹配的 `设值` 或 `当文本变化`；
- `麦金塔容器视图`：提供 `添视图` 和 `设间距`；
- `麦金塔自定义视图`：承载任意 `kind` 的低层兼容扩展。

通用方法：

- `添修饰器（修饰器）`
- `内边距（大小）`
- `字体（名称，字重）`
- `无障碍标签（标签）`
- `框架（宽，高）`
- `帮助（说明）`
- `禁用（值）`
- `添子（视图）`
- `描述（）`
- `JSON（）`

`设（名称，值）`、`设样式（名称，值）`、`设事件（事件名）` 继续保留，用于兼容 0.3 和尚未对象化的宿主扩展。常规业务代码应优先使用具体类方法。

修饰器对象：

- `内边距（大小）` 创建 `麦金塔内边距修饰器`；
- `字体（名称，字重）` 创建 `麦金塔字体修饰器`；
- `视图框架（宽，高）` 创建 `麦金塔框架修饰器`。

修饰器可以通过 `视图.添修饰器（修饰器）` 组合；视图上的同名便捷方法创建并添加对应对象。

控件工厂：

- `文本（内容）`
- `标题（内容）`
- `按钮（标题，事件名）`
- `文本框（标识，占位，值）`
- `安全文本框（标识，占位）`
- `多行文本（标识，值）`
- `开关（标识，标题，值）`
- `图片（系统名）`
- `列表（标识，项目）`
- `进度（值）`
- `绑定文本（绑定）`、`绑定进度（标识，绑定）`
- `绑定文本框`、`绑定安全文本框`、`绑定多行文本`、`绑定开关`
- `选项（标题，值）`、`选择器（标识，标题，选项列，绑定）`
- `滑块`、`步进器`、`日期选择器`、`颜色选择器`
- `搜索框（标识，提示，绑定）`
- `标签（标题，系统名）`、`链接（标题，地址）`
- `菜单选项（标题，动作）`、`菜单按钮（标识，标题，项目）`
- `选择列表（标识，选项列，绑定）`
- `工作表`、`浮出`、`警告`
- `检查器（标识，显示绑定，内容，检查器内容）`

布局工厂：

- `垂直（子视图）`
- `水平（子视图）`
- `滚动（子视图）`
- `分组（子视图）`
- `表单（子视图）`
- `分栏（边栏，内容，详情）`
- `标签页（项目）`
- `分隔线（）`

- `搜索（标识，提示，绑定，内容）`，使用 SwiftUI `.searchable`
- `导航目的地`、`导航栈`，路径使用选择 Binding

## 异步请求

请求是对象，不是无类型事件字典：

- `打开请求（标识，内容类型，允许多个）`；`作为导入（）` 切换为导入语义；
- `保存请求（标识，文件名，内容，内容类型）`；支持 `作为导出（）` 和 `使用Base64（）`；
- `窗口请求（标识，窗口标识，打开）`；
- `设置请求（标识）`；
- `文档请求（标识，场景标识，保存，文件名，内容，内容类型）`；`作为新建（）` 创建新文档窗口。

调用 `请求.完成时（处理器）` 注册一次结果处理器，再交给 `会话.执行请求`。`麦金塔请求结果` 提供 `请求标识`、`类型`、`已取消`、`值` 和 `错误`。打开结果的 `files` 包含文件名、路径、内容、编码和可用的 security-scoped bookmark；每个文件上限为 16 MiB。

通用系统请求也使用相同的完成协议：

- `安全存储请求（标识，动作，服务，账号，值）` 的动作可为 `get`、`set`、`delete`，数据存入系统钥匙串；
- `定时器请求（标识，启动，间隔秒）` 启动重复定时器，触发 `timer.fired` 应用事件；停止请求可通过 `.停止目标（目标标识）` 指定已启动的定时器。

## 动作与事件

`动作（名称）` 创建 `麦金塔动作`。动作表示应用中的稳定用户意图，可同时交给按钮、菜单、工具栏和输入控件。其名称推荐采用 `领域.动作`，例如 `project.open`、`draft.save`。

`事件（名称，载荷）` 创建 `麦金塔事件` 对象。事件名采用同一套标识校验规则，载荷为字典。

动作是发出事件前的配置对象，事件是原生宿主回传后的运行时对象。`应用.当事件（处理器）` 注册唯一应用级处理器。绑定输入未显式配置动作时使用 `binding.changed`，载荷包含 `source`、`binding`、`revision` 和类型化 `value`；兼容的非绑定输入仍发送 `source` 与 `value`。

处理器执行在言序 VM 所有者线程。原生层先投递到有界事件队列，再调用宿主事件泵，不会从任意 AppKit 后台线程直接重入言序。
