# 成熟麦金塔应用指南

成熟应用不是控件堆叠，而是一组稳定习惯：窗口可信、命令可预期、状态可恢复、键盘可抵达、错误可解释。

## 结构

建议从三层组织应用：

- 应用层：名称、菜单、设置、文档型标记；
- 窗口层：尺寸、工具栏、默认命令、根布局；
- 视图层：内容、控件、无障碍、帮助说明和禁用态。

`项目工作台.yx` 展示开发者工具的三栏结构；`写作发布器.yx` 展示内容应用的草稿、编辑、预览结构。

## 命令

把主要动作放进菜单，再把高频动作放进工具栏。事件名保持稳定，例如：

```text
project.open
draft.save
publish.start
build.clean
```

稳定事件名是测试、日志、遥测和快捷键的共同语言。

## 设置

设置页使用 `设设置（视图）` 挂到应用描述上。密钥、令牌、密码类输入使用 `安全文本框`。设置的持久化不在 0.1 版描述层直接处理，建议由业务层统一保存。

## 无障碍

列表、图标按钮、图像和复杂布局应补 `无障碍标签`。无法从标题推断用途的控件应补 `帮助`。

## 原生感

麦金塔应用应优先使用：

- 系统菜单和键盘快捷键；
- 工具栏承载主要动作；
- 分栏组织复杂信息；
- 表单承载设置；
- 明确的最小窗口尺寸；
- 进度与禁用态表达等待和不可用。

## 后续扩展建议

0.1 版已经建立应用形状。面向生产应用，下一步通常会补：

- 状态绑定与变更事件；
- 文件打开、保存、最近项目；
- 偏好设置持久化；
- 系统通知、分享、打开面板；
- 文档型应用生命周期；
- 更完整的菜单栏挂载；
- 自动化测试快照。
