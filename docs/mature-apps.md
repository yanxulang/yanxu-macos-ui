# 成熟麦金塔应用指南

成熟应用不是控件堆叠，而是一组稳定习惯：窗口可信、命令可预期、状态可恢复、键盘可抵达、错误可解释。

## 结构

建议从三层组织应用：

- 应用层：名称、运行会话、菜单、设置与文档场景；
- 窗口层：尺寸、工具栏、默认命令、根布局；
- 视图层：内容、控件、无障碍、帮助说明和禁用态。

`项目工作台.yx` 展示开发者工具的三栏结构；`写作发布器.yx` 展示内容应用的草稿、编辑、预览结构。

## 命令

把主要动作放进菜单，再把高频动作放进工具栏。事件名保持稳定，例如：

```text
project.open
draft.save
publish.start
build.clean
```

稳定事件名是测试、日志、遥测和快捷键的共同语言。

## 设置

设置页优先使用 `设置场景（视图）` 并通过 `应用.添场景` 挂载。宿主提供标准 Settings 菜单和原生窗口；密钥、令牌、密码类输入使用 `安全文本框`。设置持久化仍由言序业务层统一管理。

## 无障碍

列表、图标按钮、图像和复杂布局应补 `无障碍标签`。无法从标题推断用途的控件应补 `帮助`。

## 原生感

麦金塔应用应优先使用：

- 系统菜单和键盘快捷键；
- 工具栏承载主要动作；
- 分栏组织复杂信息；
- 表单承载设置；
- 明确的最小窗口尺寸；
- 进度与禁用态表达等待和不可用。

## 文件与文档

使用 `麦金塔运行会话.执行请求` 打开系统文件面板。打开、保存、导入、导出和文档请求都按稳定 ID 回传 `麦金塔请求结果`；不要从通用事件载荷里猜测请求来源。文档场景是可重复实例化的窗口模板，每个实例拥有独立 Store；用 `绑定内容` 与 `绑定路径` 指定打开时初始化的文字状态，文件格式仍由应用定义。

## 后续扩展建议

0.3 版已经建立事件与状态更新闭环。面向生产应用，下一步通常会补：

- 文件打开、保存、最近项目；
- 偏好设置持久化；
- 系统通知与分享；
- 文档协调读取、版本冲突和自动保存；
- 自动化测试快照。
