# yanxu-macos-ui

用言序编写 macOS 原生界面和应用逻辑。

`yanxu-macos-ui` 将言序构建的声明式应用快照交给 SwiftUI 渲染，并通过 AppKit 管理窗口、菜单和工具栏。控件事件经言序原生 ABI v2 回到应用处理器；处理器修改模型并提交新快照后，现有窗口会原位更新。整个业务层可以保留在言序中，不需要 WebView，也不要求应用项目包含 Swift 源码。

- 当前版本：`0.4.0`
- 运行环境：macOS 13 或更高版本、Apple Silicon（arm64）、言序 1.1.7 或更高版本

## 安装

```sh
yanbao add macos-ui
```

在应用中导入：

```yanxu
引「包:macos-ui」为 界面；
```

顶层应用必须显式允许加载经过锁文件校验的原生扩展：

```toml
[权限]
原生扩展 = true
```

包内已经包含 macOS arm64 动态库。普通应用不需要安装 Swift 工具链，也不需要配置外部 runner。

## 最小交互示例

```yanxu
引「包:macos-ui」为 界面；

定 执行动作 为 界面.动作（「task.run」）；
定 状态文本 为 界面.文本（「尚未执行」）；
定 根视图 为 界面.垂直（【
    界面.标题（「言序原生应用」）,
    状态文本,
    界面.按钮（「执行」，执行动作）
】）；
根视图.内边距（20）；

定 主窗口 为 界面.窗口（「言序」，根视图）
    .尺寸（720，480）
    .最小尺寸（480，320）；

定 应用 为 界面.应用（「言序」）.添窗口（主窗口）；

法 处理事件（所事件）则
    若 所事件.名称 等于「task.run」则
        状态文本.设内容（「已在言序侧处理」）；
        界面.更新（应用）；
    终
终

应用.当事件（处理事件）；
界面.运行（应用）；
```

这段代码完成一个完整闭环：SwiftUI 按钮产生事件，ABI v2 将事件投递到言序 VM，言序处理器更新模型，再由 `界面.更新（应用）` 刷新正在显示的窗口。

## 运行模型

1. 言序对象生成 `dev.yanxu.mac-ui.v1` JSON 应用快照。
2. `界面.运行（应用）` 装载清单锁定并校验过的 ABI v2 动态库。
3. Swift 宿主使用 SwiftUI 渲染视图，并用 AppKit 管理应用事件循环。
4. 按钮、菜单、工具栏和已注册的输入事件通过有界队列回到 `应用.当事件` 处理器。
5. 处理器修改窗口或视图对象，并调用 `界面.更新（应用）` 提交下一份完整快照。
6. 最后一个窗口关闭后，原生事件循环退出，回调句柄按 ABI 生命周期释放。

事件处理器始终在言序 VM 所有者线程执行。原生 UI 线程不会绕过事件队列直接重入言序代码。

## 对象模型

0.4 起，工厂函数是具体对象的便捷构造入口，而不是把所有控件降格为同一种字典：

```text
麦金塔视图
├── 麦金塔文本视图
├── 麦金塔交互视图
│   ├── 麦金塔按钮视图
│   └── 麦金塔输入视图
│       ├── 麦金塔文本框视图
│       ├── 麦金塔多行文本视图
│       └── 麦金塔开关视图
└── 麦金塔容器视图
    ├── 麦金塔垂直视图
    ├── 麦金塔水平视图
    ├── 麦金塔滚动视图
    └── 麦金塔表单视图
```

具体类提供与自身语义匹配的方法，例如 `文本.设内容`、`文本框.当文本变化`、`开关.设值`、`容器.添视图`。`麦金塔动作` 表达可复用的用户意图，`麦金塔视图修饰器` 对应 SwiftUI modifier 的对象边界。`kind` 和 property bag 仍存在于 JSON ABI 内部，用来保持 Swift 宿主兼容，不再是推荐的应用编程模型。

## 当前能力

| 领域 | 状态 | 说明 |
| --- | --- | --- |
| 窗口 | 已实现 | 多窗口、初始尺寸、最小尺寸、可调整大小、标题更新 |
| 原生菜单 | 已实现 | 菜单、命令、快捷键和统一事件回传 |
| 工具栏 | 已实现 | 原生 `NSToolbar` 项和事件回传 |
| 基础视图 | 已实现 | 文本、标题、按钮、SF Symbols 图片、分隔线、间隔、进度 |
| 输入控件 | 已实现 | 文本框、安全文本框、多行文本、开关；可发送类型化 `value` |
| 布局 | 已实现 | 垂直、水平、滚动、分组、表单、三栏分栏 |
| 集合视图 | 基础实现 | 只读列表；标签页当前使用简单文字项目 |
| 视图状态 | 已实现 | 本地编辑状态与应用快照合并；业务显式修改的值优先 |
| 通用体验 | 已实现 | 禁用态、帮助文字、无障碍标签；有限字体、字重和内边距样式 |
| 设置页、文档型应用 | 仅协议建模 | 描述字段存在，尚未接入完整 AppKit/SwiftUI 生命周期 |
| 强调色、命令角色、默认命令 | 仅协议建模 | 当前宿主尚未应用这些字段的系统行为 |

输入控件使用动作对象注册变更事件。载荷至少包含控件标识与当前值：

```yanxu
定 标题变化 为 界面.动作（「title.changed」）；
定 标题输入 为 界面.文本框（「title」，「标题」，「未命名」）
    .当文本变化（标题变化）；

法 处理事件（所事件）则
    若 所事件.名称 等于「title.changed」则
        置 主窗口.标题 为 所事件.载荷【「value」】；
        界面.更新（应用）；
    终
终
```

## 明确边界

- 本项目专注 macOS，不计划用同一宿主抽象其他桌面平台。
- 当前发布制品仅提供 arm64；Intel macOS 尚无预编译动态库。
- `更新` 以完整应用快照为一致性边界，目前不是字段级 diff 或 SwiftUI `Binding` 的跨语言直接映射。
- Swift 渲染器只支持能力表中的视图种类。未知 `kind` 会显示为不支持的占位文本。
- Swift 宿主使用通用 property bag 解码传输描述，因此增加属性不需要修改固定 schema；公开 API 则优先通过具体类和方法约束语义。增加新的原生控件仍需实现 SwiftUI 渲染逻辑。
- 文件、网络、持久化、通知、打开面板和沙箱权限属于应用业务层或后续原生能力，不由 UI 描述自动获得。
- 当前宿主一次只运行一个活动应用会话。

## 架构

```text
言序应用模型
  -> JSON 应用快照
  -> ABI v2 动态库
  -> SwiftUI 视图树 + AppKit 应用设施
  -> callback_post / owner-thread pump
  -> 言序事件处理器
  -> 下一份应用快照
```

主要模块：

- `src/言序麦金塔界面.yx`：保持兼容的中文公共门面；
- `src/动作.yx`、`src/修饰器.yx`：用户意图值对象和类型化 modifier 策略；
- `src/视图.yx`：视图继承树、具体控件与布局容器；
- `src/应用模型.yx`：事件、窗口、命令、菜单和应用聚合根；
- `src/运行时.yx`：原生宿主所有权、事件转发和快照提交；
- `src/基础.yx`：跨领域共享的校验与防御性复制；
- `YanxuMacUIModel.swift`：应用快照与通用视图属性解码；
- `YanxuMacUIRenderer.swift`：SwiftUI 递归渲染和输入状态；
- `YanxuMacUIAppHost.swift`：`NSApplication`、窗口、菜单和工具栏生命周期；
- `YanxuMacUICallback.swift`：ABI v2 回调保留、投递、泵送和释放；
- `YanxuMacUIExportsV2.swift`：`validate`、`run`、`update` 原生入口。

更完整的线程、所有权和安全边界见[架构与原生宿主](docs/architecture.md)。

## 开发与验证

在仓库根目录运行：

```sh
yanbao check
yanbao audit --offline
yanxu 查 tests/工具链.yx
swift test --package-path native
swift build -c release --package-path native
```

修改原生宿主并替换发布动态库后，必须同步更新 `言序.toml` 中的文件大小和 SHA-256，并重新生成 `言序.lock`。清单摘要不匹配时，言序运行时会拒绝加载制品。
