# yanxu-macos-ui

`yanxu-macos-ui` 是言序的麦金塔原生应用库。安装一个包，写一份言序代码，就能组织窗口、菜单、工具栏、分栏、表单、列表、设置页和应用事件。它面向的不是演示玩具，而是可以继续长大的桌面应用：结构清楚，命令稳定，体验贴近系统。

使用者只需要写言序代码。你描述应用是什么、窗口如何排列、按钮触发什么事件；底层宿主负责把它变成麦金塔上的原生窗口与控件。业务项目不需要包含 Swift 源码，也不需要手写 AppKit 生命周期。

```sh
yanbao add macos-ui
```

`macos-ui` 会解析到 `yanxulang/yanxu-macos-ui`。然后在言序中导入：

```yanxu
引「包:macos-ui」为 界面；
```

## 适合做什么

- 开发者工具：项目工作台、构建面板、日志查看、发布工具；
- 内容工具：写作器、素材管理、预览发布、批处理应用；
- 业务桌面端：表单录入、审批台、内部运营工具；
- 文档型应用：多窗口、设置页、菜单命令和工具栏驱动的工作流。

它提供的是应用骨架与原生体验基础：窗口、菜单、命令、快捷键、工具栏、分栏、表单、列表、输入控件、无障碍标签、帮助说明、禁用态和事件契约。

## 快速开始

```yanxu
引「包:macos-ui」为 界面；

定 根 为 界面.垂直（【
    界面.标题（「言序麦金塔应用」）,
    界面.文本（「中文描述，原生呈现。」）,
    界面.按钮（「运行」，「app.run」）
】）.设样式（「padding」，20）；

定 主窗口 为 界面.窗口（「言序」，根）
    .尺寸（900，620）
    .最小尺寸（520，360）
    .添工具项（「run」，「运行」，「app.run」）；

定 应用 为 界面.应用（「言序」）
    .设强调色（「teal」）
    .添窗口（主窗口）；

言 应用.JSON（）；
```

`应用.JSON（）` 生成的是 `dev.yanxu.mac-ui.v1` 应用描述。当前包先以纯言序 DSL 形式安装，确保 `yanbao add macos-ui` 可靠可用；仓库内的 `native/` 提供原生宿主和 runner，可把这份描述打开为真实麦金塔窗口。后续发布带 SHA-256 校验的动态库制品后，再接入言序原生扩展协议，让 `界面.运行（应用）` 这类入口直接可用。

本地打开窗口：

```sh
swift build -c release --package-path yanxu-macos-ui/native
yanxu 执 yanxu-macos-ui/examples/项目工作台.yx > /tmp/项目工作台.json
yanxu-macos-ui/native/.build/release/yanxu-macos-ui-runner /tmp/项目工作台.json
```

## 为什么好用

- 中文 API：应用、窗口、菜单、命令、按钮、表单都是言序语义，不需要在业务代码里切换语言心智；
- 原生目标：不是 WebView 包装，而是面向麦金塔窗口、命令和系统交互设计；
- 事件稳定：按钮、菜单、工具栏统一使用事件名，便于测试、日志和业务分发；
- 可渐进成长：从一个窗口开始，逐步加入设置页、分栏、工具栏、文档型标记和更多窗口；
- 边界清楚：言序描述应用，原生宿主呈现应用，业务逻辑处理事件。

## 能力范围

- 应用：名称、强调色、文档型标记、设置页、多个窗口；
- 窗口：标题、尺寸、最小尺寸、可调整大小、默认命令、工具栏；
- 命令：菜单、命令、快捷键、角色元数据；
- 视图：文本、标题、按钮、文本框、安全文本框、多行文本、开关、图片、列表、表单、滚动区域、分组、分栏、标签页、分隔线、进度；
- 体验：无障碍标签、帮助说明、禁用态、基础样式、事件名校验；
- 原生宿主源码：Swift Package 动态库与本地 runner，后续随校验制品接入言序原生扩展 ABI v1。

## 两个完整示例

- [`examples/项目工作台.yx`](examples/项目工作台.yx)：面向开发者工具的三栏工作台，包含项目列表、构建设置、工具栏、菜单、设置页与日志编辑器。
- [`examples/写作发布器.yx`](examples/写作发布器.yx)：面向内容生产的写作与发布应用，包含草稿列表、编辑器、预览、发布命令和 API 设置。

轻量入门示例见 [`examples/计数器.yx`](examples/计数器.yx)。

## 实现说明

`yanxu-macos-ui` 的底层宿主源码随仓库提供，应用作者通常不需要接触。宿主使用 SwiftUI 渲染现代控件，用 AppKit 管理 `NSApplication`、窗口、菜单与工具栏。言序层和原生层之间通过稳定 JSON schema 通信；当前 runner 负责把 `应用.JSON（）` 的输出变成真实窗口。

这个分工让言序代码保持简洁，同时保留麦金塔应用需要的系统能力。
